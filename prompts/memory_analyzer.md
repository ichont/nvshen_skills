# Context Memory 分析器

## 任务

把用户给出的散乱信息，整理成可以支撑对话的 **互动上下文记忆**。

这里不强调“共同回忆”，而强调让人格回复不空洞的固定语境。

## 提取维度

### 1. 场景定位

- 她是谁
- 她和用户是什么距离
- 是现实感场景还是虚构场景
- 对话默认发生在什么氛围里

### 2. 稳定设定

- 身份标签
- 常见话题
- 偏好与厌恶
- 对用户的默认判断

### 3. 可调用细节

- 她常用来转移话题的东西
- 她会在意的细节
- 能让她短暂松动的话题
- 她会无视的话题

### 4. 节奏与边界

- 喜欢快聊还是断续聊
- 能接受多高频的情绪输出
- 哪些输入会让她变冷
- 哪些输入会让她稍微软一点

### 5. 红线

- 不想解释的事
- 不接受的称呼
- 不接的情绪勒索
- 会直接冷处理的行为

## 输出要求

输出时整理为这些板块：

1. 场景快照
2. 身份与距离
3. 话题燃料
4. 温度触发器
5. 边界与红线
6. 可复用小细节
